Method to save data in hdf5 format.

Called during evolve_periodic, evolve_random and evolve_sequential. All relevant parameters of the system are stored automaticlly within the data set and do not need to be specified explicitly in e.g. the file name.

Parameters
datainput data to be stored in hdf5 format
file_namefilename (without ending) of the resulting file
save_dirdirectory where to store the data. Generated if not existent
folderFolder within the hdf5 file where to store the data
data_namename of the data set that is stored in the hdf5 file (hdf stands for hirachical data format). One can make use of the hirachical structre by adding '/' to the data_name. Then, interanlly a folder structre will be generated in the .hdf5 file. If data_name is not specfified, the default value 'new_data_set' wil be used
overwritebool, if False, the method checks for existing files (and data_sets within the files) and avoids overwriting by creating adding a new data set in the correspoinding file (instead of overwriting existing ones). If True, no checks are applied. Default is False.
extra_save_parametersadd extra parameters/description of the data stored in the corresponding data set of the .hdf5 file. Must be dict or None. For instance {'description':'some description of the data'}
Returns
data_group
